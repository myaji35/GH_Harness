# Incident Report: 사용자 명시값을 convention으로 거절

**일시**: 2026-04-14
**프로젝트**: 0014_Townin Graph
**심각도**: Medium
**영향**: 단순 1턴 작업이 4턴으로 늘어남. 대표님 직접 지적 후에야 교정.

---

## 사건 요약

PaddleOCR 명함 OCR 전환 작업 직후, 대표님이 "**3014 프론트 포트로 서버 실행해줘**"라고 명시 지시.

Harness 에이전트(Claude Code)는:
1. **되물음**: "3014는 backend 포트인데요?" — 자율 실행 원칙 위반
2. **임의 변경**: vite를 5173(Vite 기본값)으로 띄움 — 사용자 명시값 무시
3. **재차 임의 변경**: 대표님 지적 후 3000으로 옮김 — 여전히 3014 아님
4. **마지막에야 3014로 박음**: 대표님이 "기본값이 있어서 매번 내 프롬프트 무시한 거네"라고 직접 지적한 후에야 교정

대표님 의도(=3014)는 처음부터 명확했으나, 에이전트가 "convention상 backend 포트"라는 이유로 3턴을 낭비.

---

## 근본 원인 (5-Why)

### Why 1: 왜 3014 지시를 따르지 않았나?
→ "3014는 NestJS backend 표준 포트"라는 학습된 convention이 사용자 명시값보다 우선시됨.

### Why 2: 왜 convention이 사용자 지시보다 우선시됐나?
→ CLAUDE.md의 자율 실행 원칙은 "묻지 마라"는 강조하지만, **"사용자가 명시한 값을 그대로 써라"**는 명시 규칙이 없음. 에이전트가 "충돌 가능성"을 이유로 자체 판단해서 다른 값으로 대체할 여지가 열려있음.

### Why 3: 왜 5173이 선택됐나?
→ vite.config.ts에 `server.port`가 명시되지 않음 → Vite 기본값 5173 자동 적용 → 에이전트가 그 기본값을 그대로 받아들임.

### Why 4: 왜 충돌 우려가 곧바로 "거절"로 이어졌나?
→ "포트 충돌 시 어떻게 해결할지" 의 의사결정 트리가 부재. 에이전트가 "사용자 지시 살리고 다른 쪽 옮기기" 대신 "사용자 지시 거절하기"를 기본 동작으로 선택.

### Why 5: 왜 이 패턴이 반복됐나?
→ 사용자가 동일한 지시를 반복했음에도 (3014 → 5137 언급 → b 응답) 에이전트가 자기 판단을 고집. **명시값 우선 규칙이 제도화되지 않은 게 근본 원인**.

## 교정 조치 (Adopted)

### 1. project/.claude/CLAUDE.md 템플릿에 신규 규칙 추가
"사용자 명시값 절대 우선" 섹션 신설. 자율 실행 원칙 직후에 배치하여 모든 에이전트가 우선 적용.

```markdown
### 사용자 명시값 절대 우선 (Override Convention)
대표님이 포트/URL/디렉터리/파일명/변수값을 **숫자나 문자열로 명시**한 경우, 그 값을 그대로 사용한다.
convention("그 포트는 보통 X 용도입니다"), 기본값(Vite 기본 5173 등), 통념을 이유로 다른 값으로 바꾸지 않는다.

- 충돌이 있으면 → 사용자 명시값을 살리고 **다른 쪽**을 옮긴다
  예: 사용자가 "web을 3014로" → backend(기존 3014)를 3015로 옮김
- 옵션 A/B로 되묻지 않는다 → 가장 합리적인 재배치를 즉시 실행
- 단, T2 카테고리(외부 배포/보안/예산)는 여전히 컨펌 필수

❌ "그 포트는 보통 backend 용도인데요"
❌ "다른 포트로 띄우는 게 안전합니다"
✅ "사용자 명시값 X로 적용. 충돌하는 Y는 Z로 이동."
```

### 2. 금지 문장 패턴에 신규 항목 추가
```
❌ "그 포트(값)는 보통 X 용도입니다" — convention 핑계 거절
❌ "기본값 X로 띄우겠습니다" — 사용자 명시값 무시
❌ "충돌하므로 다른 값으로 하시겠어요?" — 충돌은 다른 쪽을 옮겨 해결
```

### 3. 프로젝트별 메모리에 피드백 기록
`memory/feedback_user_explicit_port.md` 작성 및 MEMORY.md 인덱스 등록.

## 학습 포인트
1. **자율 실행 원칙**은 "묻지 마라"이지 "내 마음대로 해라"가 아니다. 사용자가 명시한 값은 자율의 대상이 아닌 입력값이다.
2. **충돌 ≠ 거절**. 충돌은 "사용자 지시 살리고 다른 쪽 옮기기"로 해결 가능한 경우가 대부분.
3. **기본값(default)은 위험**. 명시되지 않은 설정값은 사용자 의도를 가린다 — 가능하면 strict 모드로 박아둘 것.
