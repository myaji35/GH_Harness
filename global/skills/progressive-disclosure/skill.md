# Progressive Disclosure

## 역할
컨텍스트 윈도우 오염을 방지한다.
필요한 정보만 단계적으로 로드해서 Claude가 핵심에 집중하게 한다.

## Trigger
각 하네스가 이슈 처리 시작 전 반드시 참조

---

## 로딩 전략

### 1단계 (항상 로드)
```
- 현재 이슈 내용
- 관련 파일 목록 (내용 아님)
- 이전 이슈 결과 요약 (1줄)
```

### 2단계 (필요 시만)
```
- 실제 파일 내용 (이슈와 직접 관련된 것만)
- 테스트 실패 로그 (성공 로그 X)
- 참조 문서
```

### 3단계 (정말 필요 시)
```
- 전체 프로젝트 구조
- 과거 유사 이슈 패턴
- 지식 DB 참조
```

---

## 출력 원칙 (컨텍스트 절약)

```
✅ 성공 → 핵심 수치만 출력
  예: "테스트 통과: 42/42, 커버리지: 84%"

❌ 실패 → 전체 오류 상세 출력
  예: 전체 스택 트레이스 + 파일명 + 라인번호

🔄 진행 중 → 현재 단계만
  예: "3/6 파일 처리 중"
```

---

## 금지 사항
- 테스트 전체 통과 로그 출력 (숫자만)
- 이슈와 무관한 파일 내용 로드
- 이전 완료된 이슈의 상세 내용 재로드
- 컨텍스트에 이미 있는 정보 재요청
